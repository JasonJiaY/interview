1. 介绍一下函数原型上bind、apply和call这 3 个方法，它们之间有什么异同。(初/中级)

共同点：这 3 个方法，在调用时，传入的第一个参数，都用于改变函数内部的this指向当前参数。

不同点：：bind(thisArg, arg1, arg2, ...argN)调用时，并没有执行原函数，而是返回一个改变原函数内部this指向和设置默认参数的新函数。apply和call，功能一致，都是改变函数内部this指向，并执行函数；不同的是传参不一致：func.apply(thisArg, arg1, arg2,...argN) ，func.apply(thisArg, [arg1, arg2,...argN])

2. 实现一个Function.prototype.bind。(中级)

// args为默认参数
Function.prototype.myBind = function (thisArg, ...args) {
  const fn = this;
  return function (...argOthers) {
    return fn.call(thisArg, ...args, ...argOthers);
  };
};

4. 介绍一下防抖和节流的原理、应用和代码实现。(中/高级)

防抖 debounce

应用

高频触发的场景，防止连续触发。 1.搜索框等 当用户在搜索框输入停止超过一段时间后，才正真发起搜索请求。 2.滚动、滑动页面停止后处理。

原理

设置一个间隔执行函数，用户连续输入时，取消上一次设置的间隔函数，并设置一次新的间隔执行函数。

实现

function debounce(fn, wait) {
  let timeoutId;
  return function (...args) {
    // 清除之前的间隔函数
    clearTimeout(timeoutId);
    timeoutId = setTimeout(() => {
      fn(...args);
    }, wait);
  };
}

节流 throttle

应用

1.短信验证码倒计时。 一段时间内，只允许用户触发一次请求。

原理

记录上一次事件触发的时间，必须在间隔多少时间后，才允许该事件再次触发。

实现

function throllte(fn, wait) {
  let prev = 0;
  return function (...args) {
    const now = Date.now();
    // 时间不满足
    if (prev + wait > now) return;
    prev = now;
    fn(...args);
  };
}

6. new 关键字的作用，做了哪些事，自己实现一个。(中级)

答案

使用new关键字来创建一个对象。 js 中创建对象，分以下步骤： 1.创建一个空对象。 2.链接该对象到另一个对象。 3.将这个对象作为上下文。 4.返回这个对象。

实现

function Person(name, age) {
  this.name = name;
  this.age = age;
}
Person.prototype.say = function () {
  console.log(this.name, this.age);
};
const p = new Person("Jason", 25);
p.say();

// 实现
// 调用方式：myNew(Person, 'Jason', 25);
function myNew(clazz, ...args) {
  const instance = {};
  clazz.call(instance, ...args);
  instance.__proto__ = clazz.prototype;
  return instance;
}

7. 说一说箭头函数与普通函数的区别。(初/中/高级)

答案

  1. 首先是写法不一样，普通函数使用function关键字声明，箭头函数没有关键字而是=>。

    // 普通函数
function fn() {}
// 箭头函数
const fn = () => {};

  2. 最重要的是this指向的问题，箭头函数中的this指向其所在的上下文的this；而普通函数有自己单独的this指向。

  3. 箭头函数不能作为构造函数，不能使用new关键字实例化对象。
  4. 箭头函数也没有自己的arguments，在箭头函数中访问arguments指向的是其外层上下文中的arguments。
  5. bind、call、apply对箭头函数无效。
  6. 箭头函数不能用作 Generator。不能使用yield关键字。

8. 有了解过Object.freeze()方法吗？冻结一个对象，会有哪些作用或影响。(中级)

答案

冻结一个对象。被冻结的对象就不再能够修改。不能给这个对象新增属性、删除属性和修改属性的值以及可枚举性、可配置性和可写性。另外，冻结后的对象，其原型也不能被修改。 注意，如果这个对象的某个属性是一个对象，那么这个属性的属性是可以修改的，除非它也是被冻结的。 数组也是一种对象。被冻结后，不能够修改其元素，不能新增或删除。

9. 有没有方式给一个对象解冻?(不能) 若想要修改这个对象怎么办？(只能拷贝一个对象) 说一说浅拷贝&深拷贝，如何实现一个对象的深拷贝？(中/高级)

答案

1.最简单&最常用的版本

JSON.parse(JSON.stringify(obj));

不足：函数，循环引用等情况直接舍弃了。

2.基础版本

function deepClone(target) {
  if (typeof target !== "object") return target;
  // 对象或数组
  const cloneTarget = Array.isArray(target) ? [] : {};
  for (let k in target) {
    cloneTarget[k] = deepClone(target[k]); // 递归拷贝
  }
  return cloneTarget;
}

3.进一步，考虑循环引用的问题。

使用WeakMap来记录已被拷贝的对象。

function deepClone(target, map = new WeekMap()) {
  if (typeof target !== "object") return target;
  // 对象或数组
  let cloneTarget = map.get(target);
  if (cloneTarget) return cloneTarget;
  cloneTarget = Array.isArray(target) ? [] : {};
  map.set(target, cloneTarget);

  for (let k in target) {
    cloneTarget[k] = deepClone(target[k], map); // 递归拷贝
  }
  return cloneTarget;
}
