2. 了解 options 请求吗？有什么作用？是否可以省略掉 options 请求?

通常在 cors(跨域资源共享)中，使用 options 请求发起一个预检请求，以检测实际请求是否可以被服务器接受。

预检请求的报文中的Access-Control-Request-Method header 字段告知服务器实际请求所使用的的 HTTP 方法;

Access-Control-Request-Headers header 字段告知服务器实际请求所携带的自定义 header 字段。

服务器基于预检请求携带的信息来判断，是否接受接下来的实际请求。

Access-Control-Allow-Origin: http://foo.example
Access-Control-Allow-Methods: POST, GET, OPTIONS
Access-Control-Allow-Headers: X-PINGOTHER, Content-Type
